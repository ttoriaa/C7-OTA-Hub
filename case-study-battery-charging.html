Case Study

Battery & Charging Benchmark Portal

从懂车帝参数抓取到可视化发布的端到端案例。核心目标是把分散的车型和充电字段，转成可比较、可追溯、可持续更新的产品化页面。

Open Live Site Back To Home

Context

业务场景是高频更新的新能源车型参数对比。信息源字段异构、口径不一，人工整理很难持续，且难以复用到日报、洞察页和对外沟通。

Problem

  • 字段分散，缺少统一 schema，无法稳定输出横向对比。
  • 价格和补齐字段存在缺失，手工修复成本高。
  • 从数据抓取到站点更新的路径不连贯，发布不可预期。

Approach

  1. 构建 daily pipeline：抓取、清洗、字段补齐、质量检查、报告产出。
  2. 分层输出：raw data、benchmark table、dashboard-ready assets。
  3. 自动化同步：日报完成后驱动站点 data、insights、页面刷新。

Architecture

Data Ingestion

Python scripts for crawl, normalization, and unresolved price checks.

Report Layer

Daily artifacts under reports for repeatable benchmark narratives.

Site Delivery

Post-daily sync script updates static pages and deploy-ready assets.

Outcomes

1 Unified benchmark portal
Daily Refresh cadence supported
Multi Views: data, dashboard, insights
Reusable Pipeline ready for new models

Lessons

将“脚本”提升为“产品能力”的关键，不是单次抓取成功，而是把字段定义、质量校验、发布路径和叙事模板统一到可重复流程中。
